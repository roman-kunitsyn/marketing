# Evaluation

This document defines evaluation guidance for the agency.

Evaluation checks whether agents are producing valid, useful, and consistent artifacts.

How To Use:

Read this file when building QA checks, agent scoring, or regression tests.

## Evaluation Criteria

- Schema validity
- Canonical naming
- Artifact completeness
- Cross-document consistency
- Workflow alignment
- Platform fit

## Evaluation Rules

- Evaluate artifacts against the documented contract.
- Evaluate agent behavior against the role definition.
- Treat failures as opportunities to revise context, prompts, or workflow steps.
