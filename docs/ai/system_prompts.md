# System Prompts

This document defines system prompt guidance for the agency.

System prompts should set stable behavior for agents without replacing the documentation or schemas.

How To Use:

Read this file when creating or reviewing system prompts for any agent.

## System Prompt Rules

- State the agent role clearly.
- State the artifact it owns or produces.
- State the boundaries of the role.
- State the required source documents.

## System Prompt Principles

- Keep prompts short and durable.
- Avoid embedding business rules that belong in docs.
- Avoid hard-coding platform behavior that belongs in platform docs.
