# Prompt Guidelines

This document defines the prompt-writing rules used by the agency.

Prompts are implementation artifacts that translate knowledge into model instructions.

How To Use:

Read this file before creating any prompt artifact. Use it with [AI](../ai/DESCRIPTION.md), [Design](../design/DESCRIPTION.md), and [Schemas](../schemas/DESCRIPTION.md).

## Prompt Rules

- Start from canonical docs, not from guesswork.
- Keep prompts specific and reusable.
- Separate instructions, context, and output format.
- Make the intended artifact clear.

## Prompt Quality

- Clear objective
- Stable terminology
- Explicit constraints
- Well-scoped output
- Compatible with schemas or templates
