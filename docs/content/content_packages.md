# Content Packages

This document defines how content assets are grouped into a deliverable package.

It is the bridge between content strategy and individual production assets.

How To Use:

Read this file when creating or reviewing `content_package.json` and related asset groups.

## Package Purpose

- Group content assets for a campaign.
- Keep production organized around the approved strategy.
- Provide a stable handoff unit for copy, creative, and publishing work.

## Package Inputs

- Content strategy
- Campaign strategy
- Content type list
- Platform scope

## Package Outputs

- `content_package.json`
- Asset list
- Production notes
- Review-ready bundle

## Rules

- Every package should trace back to a content strategy.
- Every package should contain a clear list of assets.
- The package should not hide ownership or review steps.
