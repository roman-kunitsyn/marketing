# Schemas

This folder defines the JSON contracts for the agency.

Every important artifact exchanged between modules should have a schema.
Schemas keep the system deterministic, testable, and machine-readable.

Use this folder for:

- client interview schema
- marketing brief schema
- campaign strategy schema
- content strategy schema
- content package schema
- copy package schema
- creative package schema
- image prompt package schema
- publishing plan schema
- review report schema

Schemas should be organized by business domain, not by agent.
