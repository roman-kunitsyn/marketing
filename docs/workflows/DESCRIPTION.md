# Workflows

This folder defines how work moves through the system.

Workflows describe behavior: inputs, outputs, validations, and handoffs between artifacts and roles.
They should remain aligned with the canonical pipeline in the main docs.

Canonical flow:

Client Interview -> Marketing Brief -> Campaign Strategy -> Content Strategy -> Content Package -> Copy Package and Creative Package -> Image Prompt Package -> Publishing Plan -> Review -> Delivery

Use this folder for:

- client interview workflow
- campaign creation workflow
- content strategy workflow
- copywriting workflow
- creative design workflow
- prompt generation workflow
- image generation workflow
- review workflow
- approval workflow
- publishing workflow
- analytics workflow

Workflows are business processes, not execution code.
