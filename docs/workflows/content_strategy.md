# Content Strategy Workflow

This workflow defines how the content planning stage is completed.

It turns campaign strategy into structured content direction and a production-ready content package.

How To Use:

Read this file when planning content output for a campaign.

## Inputs

- `campaign_strategy.json`
- Content pillars
- Platform scope
- Content types

## Steps

1. Review campaign strategy.
2. Define content mix and formats.
3. Produce `content_strategy.json`.
4. Produce `content_package.json`.
5. Validate alignment with the campaign objective.

## Outputs

- `content_strategy.json`
- `content_package.json`

## Rules

- Content strategy must not contradict the approved campaign strategy.
- Content packages should be built from the approved content strategy.
- Keep platform adaptation separate from content planning.
