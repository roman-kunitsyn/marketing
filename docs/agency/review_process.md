# Review Process

This document defines how review happens across agency roles.

It keeps approval stages explicit so agents can decide what is ready to move forward.

How To Use:

Read this file when defining approval gates, QA checks, or revision loops.

## Review Stages

1. Role-level self-check.
2. Peer or upstream validation.
3. QA review.
4. Client review, if required.
5. Final approval.

## Review Rules

- Review should happen before every major handoff.
- QA is the final internal gate before delivery.
- A rejected artifact returns to its owning role.
- Revised artifacts must be revalidated before release.

## Review Criteria

- Completeness
- Canonical naming
- Schema alignment
- Consistency with the business brief
- Consistency with the approved strategy
- Platform and brand fit

## Outputs

- Approved artifact
- Revision request
- Rejected artifact with reasons
- Delivery-ready package
