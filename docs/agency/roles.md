# Roles

This document defines the canonical agency roles and what each role owns.

It is the source of truth for artifact ownership inside the people-and-handoffs layer.

How To Use:

Read this file before assigning work or designing handoffs. Use it with [Responsibilities](responsibilities.md) and [Communication](communication.md) to keep ownership clear.

## Canonical Roles

### Marketing Strategist

- Owns `marketing_brief.json`
- Owns `campaign_strategy.json`
- Normalizes the `client_interview.json` input into strategy-ready requirements

### Content Strategist

- Owns `content_strategy.json`
- Owns `content_package.json`

### Copywriter

- Owns `copy_package.json`

### Creative Designer

- Owns `creative_package.json`

### Prompt Builder

- Owns `image_prompt_package.json`

### Social Media Manager

- Owns `publishing_plan.json`

### QA Reviewer

- Owns `review_report.json`

## Role Rules

- One primary owner per artifact.
- Roles communicate through structured artifacts, not informal assumptions.
- A role should only own artifacts it can fully validate.
