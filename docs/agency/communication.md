# Communication

This document defines how agency roles communicate with each other.

Communication in this project should happen through artifacts, explicit handoffs, and written review notes.

How To Use:

Read this file when defining collaboration rules or building automated handoff logic.

## Communication Rules

- Communicate through artifacts first.
- Do not depend on unstated context in chat.
- Every handoff should name the artifact, the next owner, and the expected next action.
- Feedback should be recorded as structured notes whenever possible.

## Handoff Format

- Source artifact
- Current owner
- Target owner
- Change requested
- Approval state

## Handoff Principles

- Marketing Strategist hands off strategy artifacts to Content Strategist.
- Content Strategist hands off content artifacts to Copywriter, Creative Designer, or Prompt Builder as needed.
- Social Media Manager receives approved production artifacts for publishing preparation.
- QA Reviewer handles final validation before delivery.

## Escalation Rules

- Escalate missing inputs before production starts.
- Escalate scope changes before revision work continues.
- Escalate schema or quality failures before delivery.
