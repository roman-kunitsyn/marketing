# Quality Guidelines

This document defines the minimum quality bar for agency artifacts.

It exists so human teams and agents can make the same judgment about whether an artifact is ready.

How To Use:

Read this file before approving, publishing, or handing off any artifact.

## Quality Bar

- Correct artifact name
- Correct owner
- Correct schema or documented format
- Complete required fields
- Consistent terminology
- Clear next handoff
- No unresolved scope drift

## Content Quality

- Factual and concise
- Aligned with the approved brief and strategy
- Suitable for the intended platform and audience
- Free of duplicate or conflicting instructions

## Structural Quality

- Use canonical file names.
- Keep outputs modular.
- Avoid mixing strategy, execution, and review notes in one artifact.
- Preserve the artifact chain from the core docs.

## Acceptance Rule

- If an artifact fails any quality check, it is not ready for the next role.
- QA Review should only approve artifacts that can be delivered without ambiguity.
