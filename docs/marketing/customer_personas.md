# Customer Personas

This document defines the persona layer used by the agency.

Personas turn audience data into a usable profile for messaging and content planning.

How To Use:

Read this file when translating audience groups into campaign-ready profiles.

## Persona Structure

- Name or label
- Role or context
- Goals
- Pain points
- Objections
- Preferred content style
- Preferred platform behavior

## Persona Rules

- A persona should represent a real decision pattern, not a stereotype.
- Persona names should be reusable and easy to reference in other docs.
- Personas should support campaign strategy and content strategy, not replace them.
