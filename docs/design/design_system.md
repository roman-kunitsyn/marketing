# Design System

This document defines the design system layer used by the agency.

It describes the reusable visual rules that keep campaign assets coherent across content types and platforms.

How To Use:

Read this file before creating creative packages or visual references. Use it with [Content](../content/DESCRIPTION.md), [Marketing](../marketing/DESCRIPTION.md), and [Schemas](../schemas/DESCRIPTION.md).

## Purpose

- Provide a stable visual language.
- Keep creative decisions consistent.
- Support downstream design and prompt generation.

## Design System Components

- Color guidelines
- Typography
- Layout rules
- Composition rules
- Brand styling
- Image size conventions
- Image asset references

## Rules

- The design system should be reusable across campaigns.
- It should stay independent from any single image generator.
- It should describe visual behavior, not prompt syntax.
